Document Model Refresher

MongoDB stores data as rich documents instead of rigid rows. Collections gather related documents, and every document is encoded in BSON, which supports nested structures and binary data.

  • Document: Key-value pairs that can contain embedded objects and arrays. Each document owns an _id field for identity.
  • Collection: A logical grouping similar to a relational table but without enforced schemas, allowing different documents to have distinct fields.
  • Schema flexibility: Because fields are optional, applications can evolve quickly without expensive migrations.
{
  "_id": ObjectId("..."),
  "name": "Asha",
  "age": 29,
  "skills": ["Node.js", "MongoDB"],
  "availability": { "remote": true, "hours_per_week": 25 }
}

CRUD Operations Recapped

The JavaScript shell (or drivers) expose intuitive helpers for manipulating data. Here is a paraphrased set of commands I used frequently:

// Switch to the project database
use("portfolioDB");

// CREATE
const inserted = db.consultants.insertOne({
  name: "Ravi",
  region: "APAC",
  tech_stack: ["MongoDB", "Express"],
  active: true
});

// READ
const activeConsultants = db.consultants.find({ active: true });
db.consultants.findOne({ name: "Ravi" });

// UPDATE
db.consultants.updateOne(
  { name: "Ravi" },
  { $push: { tech_stack: "React" } }
);

// DELETE
db.consultants.deleteOne({ name: "Ravi" });

Index Patterns

Indexes are the core performance lever. By default MongoDB maintains an index on _id, but creating additional single-field or compound indexes can drop query latency from seconds to milliseconds.

Single-Field Index

Use when you routinely search or sort by one field.

db.consultants.createIndex({ region: 1 });

db.consultants.find({ region: "EMEA" });

Compound Index

Order matters: MongoDB will scan by the first field and then refine by subsequent fields.

db.consultants.createIndex({ active: 1, "skills.level": -1 });

db.consultants.find({ active: true }).sort({ "skills.level": -1 });

Aggregation Pipeline Snapshot

The aggregation framework allows multi-stage analytics directly inside MongoDB. The pipeline below filters consultants, groups them by region, and ranks those regions by average availability.

db.consultants.aggregate([
  {
    $match: { active: true }
  },
  {
    $group: {
      _id: "$region",
      avgHours: { $avg: "$availability.hours_per_week" },
      specialists: { $sum: 1 }
    }
  },
  {
    $sort: { avgHours: -1 }
  }
]);

Executing a pipeline like this keeps heavy lifting within the database engine, minimizing data transfer to the application tier.

Scenario Overview

This Hive example shifts to a fictional marketplace that fulfils gadget orders through both online and in-store channels. The goal is to load transactional data, combine it with a product catalog, and surface category-level revenue metrics—all while keeping the descriptions richer than before.

1. Defining Tables and Loading Data

Two external CSV files serve as the foundation. The schema definitions below enable Hive to parse them correctly:

-- Orders fact table
CREATE TABLE orders (
    order_id INT,
    customer_id INT,
    product_id INT,
    quantity INT,
    unit_price DOUBLE,
    order_date STRING,
    fulfillment_channel STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

-- Product dimension
CREATE TABLE products (
    product_id INT,
    product_name STRING,
    category STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

LOAD DATA INPATH '/warehouse/orders.csv' INTO TABLE orders;
LOAD DATA INPATH '/warehouse/products.csv' INTO TABLE products;

2. Query Walkthrough with Outputs

SELECT with Calculated Columns

Retrieve the online orders placed on launch day and compute their line totals:

SELECT
    order_id,
    quantity * unit_price AS total_value
FROM orders
WHERE order_date = '2024-03-01'
  AND fulfillment_channel = 'Online';
order_id | total_value
---------+------------
1        | 90.00

JOIN Products to Orders

Enrich each order with descriptive product information to make reports readable:

SELECT
    o.order_id,
    p.product_name,
    p.category,
    o.quantity,
    o.quantity * o.unit_price AS total_value
FROM orders o
JOIN products p
  ON o.product_id = p.product_id
ORDER BY o.order_id;
order_id | product_name           | category     | quantity | total_value
---------+------------------------+--------------+----------+------------
1        | Smart Kettle           | Kitchen      | 2        | 90.00
2        | Espresso Machine       | Appliances   | 1        | 120.00
3        | Reusable Straw Pack    | Accessories  | 3        | 54.00
4        | Smart Kettle           | Kitchen      | 1        | 45.00
5        | Bluetooth Scale        | Wellness     | 2        | 130.00
6        | Coffee Grinder         | Kitchen      | 4        | 60.00

Aggregation and GROUP BY

Summarize performance by product category to see where the revenue lands:

SELECT
    p.category,
    SUM(o.quantity * o.unit_price) AS gross_revenue,
    COUNT(DISTINCT o.order_id) AS order_count,
    AVG(o.quantity) AS avg_units_per_order
FROM orders o
JOIN products p
  ON o.product_id = p.product_id
GROUP BY p.category
ORDER BY gross_revenue DESC;
category     | gross_revenue | order_count | avg_units_per_order
-------------+---------------+-------------+---------------------
Kitchen      | 195.00        | 3           | 2.3333333333
Wellness     | 130.00        | 1           | 2.0
Appliances   | 120.00        | 1           | 1.0
Accessories  | 54.00         | 1           | 3.0

The kitchen category leads, buoyed by repeated kettle purchases, while accessories trail with a single small order.

3. Partition Strategy

To accelerate day-specific reporting, I created a partitioned summary table. Each partition stores all metrics for a given order date and shipping region, enabling Hive to skip entire directories when queries filter by those keys.

CREATE TABLE daily_sales (
    order_id INT,
    customer_id INT,
    product_id INT,
    quantity INT,
    unit_price DOUBLE,
    fulfillment_channel STRING
)
PARTITIONED BY (
    order_date STRING,
    shipping_region STRING
)
STORED AS PARQUET;

Loading data requires specifying both partition columns explicitly:

LOAD DATA INPATH '/warehouse/regions/west/2024-03-01'
INTO TABLE daily_sales
PARTITION (order_date='2024-03-01', shipping_region='US-West');

LOAD DATA INPATH '/warehouse/regions/east/2024-03-02'
INTO TABLE daily_sales
PARTITION (order_date='2024-03-02', shipping_region='US-East');

Queries that pin down the partition values execute noticeably faster because Hive only scans relevant Parquet files:

SELECT product_id, quantity
FROM daily_sales
WHERE order_date = '2024-03-02'
  AND shipping_region = 'US-East'
  AND fulfillment_channel = 'Online';
product_id | quantity
-----------+---------
301        | 1
303        | 3
